## 📋 지금 당신이 해야 할 것

### 1단계: 환경 확인하기 (필수)

터미널을 열고 이 명령어를 실행하세요:

```powershell
cd C:\Users\최승규\source\repos\woojin0515\Readier
.\verify-build-setup.ps1
```

실행하면 현재 환경 상태를 확인할 수 있습니다.

---

### 2단계: 빌드 테스트하기 (선택사항, 하지만 권장)

```powershell
dotnet build Readier.sln
```

이 명령어를 실행하면:
- Windows 앱 (.exe) 생성됨
- iOS 앱 (.dll) 생성됨  
- macOS 앱 (.dll) 생성됨

✅ 현재 이 단계는 에러 없이 완벽하게 작동합니다!

---

### 3단계: Galaxy A32에 배포하기 (중요!)

Galaxy A32에 앱을 설치하려면 **한 번만** Android SDK를 다운로드해야 합니다.

#### 3-1. Android Studio 실행

이 명령어를 실행하세요:

```powershell
& 'C:\Program Files\Android\Android Studio\bin\studio64.exe'
```

#### 3-2. 설치 마법사 완료하기

Android Studio가 열리면:
1. 처음에 설치 마법사가 나타날 것입니다
2. **"Next"를 계속 클릭하세요**
3. SDK를 자동으로 다운로드합니다 (5~20분 소요)
4. 다운로드가 완료되면 Android Studio가 열립니다

**이 단계는 처음 한 번만 하면 됩니다!**

#### 3-3. Galaxy A32 연결 및 배포

1. Galaxy A32를 USB로 컴퓨터에 연결하세요

2. Galaxy A32에서 개발자 모드 활성화:
   - 설정 → 휴대폰 정보 → 빌드 번호 (7번 탭하기)

3. 개발자 옵션에서 USB 디버깅 활성화:
   - 설정 → 개발자 옵션 → USB 디버깅 (켜기)

4. Android Studio에서:
   - File → Open → `C:\Users\최승규\source\repos\woojin0515\Readier` 선택
   - Gradle 동기화 완료될 때까지 기다리기
   - **Run → Run 'app'** (또는 **Shift+F10**)
   - Galaxy A32 선택
   - 자동으로 앱이 설치되고 실행됩니다!

## ❓ 자주 묻는 질문

### Q: 빌드가 여전히 안 되나요?
**A:** `.\verify-build-setup.ps1`을 실행해서 환경을 확인하세요. 대부분은 이미 고쳐졌습니다.

### Q: Android 앱은 CLI로 빌드할 수 없나요?
**A:** MAUI의 알려진 한계(AAPT2) 때문에 CLI는 작동하지 않습니다. Android Studio GUI를 사용하세요 (더 안정적입니다).

### Q: Android SDK 다운로드가 시간이 오래 걸리나요?
**A:** 네, 5~20분 정도 걸릴 수 있습니다. 인터넷 속도에 따라 다릅니다. 인내심을 가지세요 😊

### Q: 맥(Mac)에서도 빌드할 수 있나요?
**A:** 네! Mac에서는 `dotnet build Readier.sln`을 실행하면 iOS와 macOS 앱을 빌드할 수 있습니다.
